> JSP 화면에서 쓰는 **EL(`${}`)** 과는 다른 것이다. SpEL은 **스프링 프레임워크 전용**
> 표현 언어로, 자바 코드/설정 안에서 동작한다. EL은 별도의 EL & JSTL 글에서 다룬다.

---

## 1. SpEL 이란?

**SpEL(Spring Expression Language)** 은 런타임에 **객체 그래프를 질의·조작**할 수 있는
스프링 전용 표현 언어다. 문자열로 쓴 표현식을 스프링이 **평가(계산)** 해서 값을 만든다.

- 스프링이 직접 만든 기능 (자바 EE의 EL과 무관).
- `@Value`, 스프링 시큐리티, `@Cacheable`, XML/자바 설정 등 **스프링 곳곳에서** 쓰인다.
- 단순 값 치환을 넘어 **연산 · 메서드 호출 · 다른 빈 참조 · 컬렉션 필터링**까지 가능.

### EL 과의 차이 (헷갈림 방지)

| | EL | SpEL |
|---|---|---|
| 소속 | JSP/JSTL 표준 | 스프링 프레임워크 |
| 문법 | `${...}` | `#{...}` |
| 사용처 | JSP 화면(View) | 스프링 자바 코드/설정 |
| 능력 | 값 꺼내기·비교·empty | 메서드 호출·빈 참조·컬렉션 연산 등 |

## 2. 동작 원리

SpEL은 "문자열 표현식 → 파싱 → 평가 → 결과값" 순서로 동작한다.

```
"1 + 2 * 3"  ──(ExpressionParser)──►  Expression 객체  ──(getValue)──►  7
                                            │
                              EvaluationContext (변수·루트객체·빈 정보)
```

| 구성요소 | 역할 |
|----------|------|
| `ExpressionParser` | 표현식 문자열을 파싱 (`SpelExpressionParser`) |
| `Expression` | 파싱된 표현식. `getValue()`로 평가 |
| `EvaluationContext` | 평가에 필요한 컨텍스트 (변수, 루트 객체, 빈 등). 보통 `StandardEvaluationContext` |

### 직접 코드로 평가해보기 (원리 확인용)

```java
ExpressionParser parser = new SpelExpressionParser();

// 1) 단순 연산
Expression exp = parser.parseExpression("1 + 2 * 3");
int result = exp.getValue(Integer.class);          // 7

// 2) 문자열 메서드 호출
String msg = parser.parseExpression("'Hello'.toUpperCase()")
                   .getValue(String.class);        // "HELLO"

// 3) 객체(루트) 프로퍼티 접근
BoardVO board = new BoardVO();
board.setTitle("공지");
String title = parser.parseExpression("title")
                     .getValue(board, String.class);  // "공지"
```

> 실무에선 이렇게 직접 파싱하기보다, 아래처럼 **애너테이션/설정 속에 표현식을 적으면 스프링이 알아서 평가**한다.

## 3. 문법 — `#{...}`

스프링 설정/애너테이션 안에서는 `#{ }` 안에 표현식을 쓴다.

### 3-1. 리터럴 · 연산

```java
#{ 100 }                 // 정수
#{ 3.14 }                // 실수
#{ 'hello' }             // 문자열 (작은따옴표)
#{ true }                // 불리언

#{ 1 + 2 }               // 산술 + - * / %
#{ 10 > 5 }              // 비교 > < == != >= <=
#{ 10 > 5 and 1 < 2 }    // 논리 and or not  (&&, ||, ! 도 가능)
#{ user.age > 18 ? '성인' : '미성년' }   // 삼항
#{ name ?: '익명' }       // 엘비스 연산자: name이 null이면 '익명'
#{ user?.address?.city }  // 안전 탐색: 중간이 null이면 NPE 대신 null
```

### 3-2. 프로퍼티 · 메서드 호출

```java
#{ someBean.name }                 // 빈의 프로퍼티(getter)
#{ someBean.calc(10) }             // 빈의 메서드 호출
#{ 'abc'.length() }                // 문자열 메서드 → 3
#{ T(java.lang.Math).random() }    // T(): 정적(static) 클래스 참조
#{ T(java.lang.Math).PI }          // 정적 상수
```

### 3-3. 다른 빈 참조

```java
#{ otherBean }                     // 빈 자체
#{ otherBean.property }            // 빈의 프로퍼티
#{ @myService.getValue() }         // @빈이름 으로 빈 참조 (스프링 시큐리티 등에서)
```

### 3-4. 컬렉션 (강력한 부분 — EL엔 없음)

```java
// .?[ ]  선택(필터): 조건 맞는 요소만
#{ users.?[age > 18] }                 // 18세 초과 유저 리스트

// .![ ]  투영(매핑): 각 요소를 변환
#{ users.![name] }                     // 이름만 뽑은 리스트

// .^[ ] / .$[ ]  첫/마지막 매칭 요소
#{ users.^[age > 18] }                 // 조건 맞는 "첫" 요소
#{ users.$[age > 18] }                 // 조건 맞는 "마지막" 요소

#{ {1, 2, 3} }                         // 인라인 리스트
#{ {'a':1, 'b':2} }                    // 인라인 맵
#{ {1,2,3}[0] }                        // 인덱스 접근 → 1
```

## 4. 실제 사용처

### 4-1. `@Value` — 설정값·표현식 주입 (가장 흔함)

```java
@Component
public class AppConfig {

    // ${} = 프로퍼티 플레이스홀더: application.properties 값 치환
    @Value("${server.port}")
    private int port;

    // #{} = SpEL: 표현식을 평가
    @Value("#{ 60 * 1000 }")               // → 60000
    private int oneMinuteMs;

    @Value("#{ systemProperties['user.name'] }")   // 시스템 속성
    private String osUser;

    @Value("#{ T(java.lang.Math).random() * 100 }")  // 정적 메서드
    private double randomScore;

    @Value("#{ otherBean.defaultPageSize ?: 10 }")  // 다른 빈 값 + 기본값
    private int pageSize;
}
```

> ⚠️ **`${...}` 와 `#{...}` 는 다르다.**
> - `${...}` : properties/yml 의 **값을 가져와 치환** (프로퍼티 플레이스홀더)
> - `#{...}` : **SpEL 표현식을 계산**
> 둘을 섞을 수도 있다: `@Value("#{'${app.list}'.split(',')}")` → 콤마 문자열을 배열로.

### 4-2. 스프링 시큐리티 (권한 표현식)

```java
@PreAuthorize("hasRole('ADMIN')")                 // 관리자만
public void deleteBoard(Long bno) { ... }

@PreAuthorize("#bno == authentication.principal.id")  // 본인 글만
public void modify(Long bno) { ... }

@PreAuthorize("hasRole('ADMIN') or #board.writer == principal.username")
public void update(BoardVO board) { ... }
```

> `#bno`, `#board` 처럼 `#파라미터명` 으로 메서드 인자를 참조하는 게 SpEL의 특징.

### 4-3. `@Cacheable` 등 캐시 키

```java
@Cacheable(value = "board", key = "#bno")          // 파라미터 bno를 캐시 키로
public BoardVO get(Long bno) { ... }

@Cacheable(value = "list", key = "#cri.pageNum + '-' + #cri.amount")
public List<BoardVO> getList(Criteria cri) { ... }
```

### 4-4. `@Scheduled`, 조건부 빈 등

```java
@Scheduled(cron = "#{@scheduleConfig.cronExpression}")  // 빈에서 cron 표현식 주입
public void job() { ... }
```

## 5. EL vs SpEL — 같은 일을 어디서 하나

| 상황 | EL (JSP 화면) | SpEL (스프링 코드/설정) |
|------|---------------|--------------------------|
| 값 출력 | `${board.title}` | (화면용 아님) |
| 설정값 주입 | (불가) | `@Value("${server.port}")` |
| 메서드 호출 | 제한적 | `#{ bean.method() }` |
| 컬렉션 필터 | 불가 | `#{ list.?[age>18] }` |
| 권한 검사 | 불가 | `@PreAuthorize("hasRole('ADMIN')")` |

핵심: **EL은 "화면에서 값 꺼내기", SpEL은 "스프링 내부에서 표현식 평가".**

---

## 6. 주의점

| 주의 | 설명 |
|------|------|
| `${}` vs `#{}` 혼동 | `${}`=프로퍼티 치환, `#{}`=SpEL 평가. 가장 흔한 실수 |
| 보안 | 사용자 입력을 SpEL로 직접 평가하면 임의 코드 실행 위험(SpEL injection). 외부 입력을 표현식에 넣지 말 것 |
| 성능 | 표현식 파싱 비용 있음. 반복 평가 시 `Expression` 객체 재사용 |
| 디버깅 | 표현식 오타는 런타임에 `SpelEvaluationException` 으로 터짐 → 표현식·빈 이름 확인 |

---

## 7. 한 줄 요약

> **SpEL = 스프링 전용 `#{}` 표현 언어.** `@Value`로 설정값 주입, 시큐리티 권한,
> 캐시 키 등 **스프링 코드/설정 안에서 표현식을 평가**한다.
> JSP 화면에서 값 꺼내는 **EL(`${}`)** 과는 별개다.
