> JSTL / EL 문법은 분량이 많아 별도의 EL & JSTL 글로 분리했다. 이 문서는 JSP 자체에 집중한다.

## 1. JSP 란?

**JSP(JavaServer Pages)** 는 HTML 안에 자바 코드를 끼워 넣어, 서버에서 **동적으로 HTML을 생성**하는 기술이다.

- 정적 HTML : 파일 내용 그대로 응답 → 항상 같은 화면.
- JSP : 요청마다 서버가 자바 로직을 실행 → DB 조회 결과 등 **그때그때 다른 HTML** 생성.

Spring MVC에서는 Controller가 만든 데이터(`model`)를 받아 화면을 그리는 **View** 역할을 한다.
(Controller → Service → Mapper 흐름은 별도의 Controller·MyBatis 글 참고)

---

## 2. 동작 원리 — JSP는 결국 "서블릿"이다

JSP는 브라우저가 직접 읽는 게 아니다. **서버에서 자바 코드(서블릿)로 변환된 뒤 실행**된다.

```
[최초 요청 시 1회]
list.jsp  ──(번역, translation)──►  list_jsp.java  ──(컴파일)──►  list_jsp.class
                                                                      │
[매 요청 시]                                                          ▼
브라우저 요청 ───────────────────────────────────────────►  .class 실행 → HTML 생성 → 응답
```

| 단계 | 설명 | 시점 |
|------|------|------|
| Translation(번역) | `.jsp` → 자바 서블릿 소스(`.java`) | 최초 1회 (또는 jsp 수정 후) |
| Compile(컴파일) | `.java` → `.class` | 최초 1회 |
| Execute(실행) | `.class` 의 `_jspService()` 실행 → HTML 출력 | **매 요청마다** |

- 그래서 JSP 안의 자바 코드는 **서버에서** 돈다. 브라우저는 결과 HTML만 받는다.
- 처음 호출이 살짝 느린 이유 = 번역+컴파일 때문. 이후엔 `.class`가 캐시돼 빠르다.
- WAS(Tomcat 등)가 이 변환/실행을 담당한다.

### 요청 한 번의 전체 흐름 (전형적인 예)

```
브라우저  GET /board/list?pageNum=3
   │
   ▼
DispatcherServlet  →  BoardController.list(cri, model)
   │                     ├ Service → Mapper → DB 조회
   │                     └ model.addAttribute("list", list)
   ▼
ViewResolver  →  /WEB-INF/views/board/list.jsp 선택
   │
   ▼
list.jsp 실행  →  model의 list를 꺼내(${list}) HTML로 렌더링
   │
   ▼
완성된 HTML  →  브라우저
```

> `/WEB-INF/views/` 아래에 두는 이유: `WEB-INF`는 브라우저가 URL로 **직접 접근 불가**한 보호 영역이라, 반드시 Controller를 거치게 강제된다.

## 3. JSP 문법 총정리

JSP 안에 쓸 수 있는 요소는 크게 **5종류**다.

| 종류 | 문법 | 용도 |
|------|------|------|
| 디렉티브(Directive) | `<%@ ... %>` | 페이지 설정 (인코딩, import, include, taglib) |
| 스크립팅(Scripting) | `<% %>` `<%= %>` `<%! %>` | 자바 코드 삽입 (요즘 지양) |
| 주석(Comment) | `<%-- --%>` | JSP 주석 |
| 액션(Action) | `<jsp:... />` | 표준 동작 태그 (include, forward, useBean 등) |
| EL / JSTL | `${ }` / `<c:... >` | 값 출력·흐름 제어 (EL & JSTL 글 참고) |

---

### 3-1. 디렉티브 `<%@ ... %>`

페이지를 어떻게 번역·컴파일할지 **WAS에 지시**하는 설정. 종류는 `page`, `include`, `taglib` 세 가지.

#### (1) `page` — 페이지 전역 설정

```jsp
<%@ page language="java"
         contentType="text/html; charset=UTF-8"
         pageEncoding="UTF-8"
         import="java.util.*, java.text.SimpleDateFormat"
         session="true"
         errorPage="error.jsp"
         isErrorPage="false"
         buffer="8kb"
         autoFlush="true"
         trimDirectiveWhitespaces="true" %>
```

| 속성 | 의미 | 기본값 |
|------|------|--------|
| `language` | 스크립팅 언어 | `java` |
| `contentType` | 응답 MIME 타입 + 문자셋 | `text/html` |
| `pageEncoding` | JSP 파일 자체의 인코딩 | ISO-8859-1 |
| `import` | 자바 import (콤마로 여러 개) | - |
| `session` | HttpSession 사용 여부 | `true` |
| `errorPage` | 예외 발생 시 이동할 페이지 | - |
| `isErrorPage` | 이 페이지가 에러 처리용인지 (`exception` 객체 사용 가능) | `false` |
| `buffer` / `autoFlush` | 출력 버퍼 크기 / 자동 flush | `8kb` / `true` |
| `trimDirectiveWhitespaces` | 디렉티브 자리의 빈 줄 제거 | `false` |

> `contentType`은 "응답을 어떻게 보낼지", `pageEncoding`은 "이 .jsp 파일을 어떻게 읽을지". 한글 깨짐을 막으려면 **둘 다 UTF-8** 로 맞추는 게 안전하다.

#### (2) `include` — 정적 포함

```jsp
<%@ include file="header.jsp" %>
```
- **번역(컴파일) 시점**에 대상 파일 내용을 **그대로 합쳐서** 하나의 서블릿으로 만든다.
- 변수/선언을 공유한다(같은 파일이 되므로). 헤더/푸터처럼 고정 조각에 적합.
- 실행 시점에 포함하는 `<jsp:include>`(액션)와 다름 → 3-5 참고.

#### (3) `taglib` — 태그 라이브러리 등록

```jsp
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
```
- JSTL 등 커스텀 태그를 쓰기 위한 선언. `prefix`는 별칭. (상세는 EL & JSTL 글 참고)

### 3-2. 스크립팅 요소 (자바 코드 삽입)

> ⚠️ **요즘은 거의 안 쓴다.** HTML과 로직이 뒤섞여 유지보수가 어렵다. 표준은 **EL + JSTL**.
> 다만 레거시 코드를 읽으려면 알아야 하므로 정리한다.

#### (1) 스크립트릿 `<% ... %>` — 자바 코드 블록

```jsp
<%
    int total = 0;
    for (int i = 1; i <= 10; i++) {
        total += i;
    }
%>
```
- 변환되면 서블릿의 `_jspService()` **메서드 내부**에 들어간다 → 지역 변수처럼 동작.

#### (2) 표현식 `<%= ... %>` — 값 출력

```jsp
<p>합계: <%= total %></p>
<p>현재 시각: <%= new java.util.Date() %></p>
```
- `out.print(...)` 로 변환된다. **끝에 세미콜론 `;` 을 붙이지 않는다.**

#### (3) 선언 `<%! ... %>` — 멤버 변수/메서드

```jsp
<%!
    private int count = 0;            // 서블릿의 멤버(인스턴스) 변수

    public int square(int n) {        // 메서드 정의
        return n * n;
    }
%>
<p><%= square(5) %></p>              <%-- 25 --%>
```
- 변환되면 서블릿 클래스의 **멤버 영역**에 들어간다(메서드 밖). 스크립트릿(메서드 안)과 위치가 다르다.
- ⚠️ 선언한 멤버 변수는 **모든 요청이 공유**(서블릿은 보통 단일 인스턴스) → 동시성 문제 주의.

#### 스크립트릿 vs 선언 — 변환 후 위치 비교

```java
public class page_jsp extends HttpServlet {
    private int count = 0;           // ← <%! %> 선언이 여기로

    public void _jspService(req, res) {
        int total = 0;               // ← <% %> 스크립트릿이 여기로
        out.print(total);            // ← <%= %> 표현식이 여기로
    }
}
```

### 3-3. 주석

```jsp
<%-- JSP 주석: 서버에서 제거됨. 응답 HTML에 안 나감 (소스보기 불가) --%>
<!-- HTML 주석: 응답에 그대로 포함됨. 브라우저 소스보기로 노출됨 -->
<% // 자바 한 줄 주석 (스크립트릿 안) %>
<% /* 자바 블록 주석 */ %>
```
- 민감한 설명은 반드시 `<%-- --%>` 로. `<!-- -->` 는 사용자에게 보인다.

### 3-4. 내장(암시) 객체 — 선언 없이 바로 쓰는 9개

JSP가 번역될 때 자동으로 만들어 주는 객체들. `<% %>` 안에서 바로 쓸 수 있다.

| 객체 | 타입 | 용도 |
|------|------|------|
| `request` | HttpServletRequest | 요청 정보·파라미터·request 스코프 |
| `response` | HttpServletResponse | 응답 제어(헤더, 리다이렉트) |
| `out` | JspWriter | 응답 본문 출력 |
| `session` | HttpSession | 세션 스코프(로그인 정보 등) |
| `application` | ServletContext | 앱 전역 스코프 |
| `pageContext` | PageContext | 페이지 컨텍스트·스코프 접근 |
| `config` | ServletConfig | 서블릿 초기화 설정 |
| `page` | Object(=this) | 현재 JSP 자신 |
| `exception` | Throwable | 예외 객체(`isErrorPage="true"` 일 때만) |

```jsp
<%
    String name = request.getParameter("name");   // 요청 파라미터 ?name=...
    session.setAttribute("user", name);           // 세션에 저장
    out.println("안녕하세요 " + name);              // 출력
%>
```
> 실무에선 위 같은 직접 접근 대신 **EL의 내장객체**(`${param.name}`, `${sessionScope.user}`)를 쓴다.

### 3-5. 액션 태그 `<jsp:... />`

XML 형식의 표준 동작 태그. **실행(런타임) 시점**에 작동한다.

#### (1) `<jsp:include>` — 동적 포함

```jsp
<jsp:include page="menu.jsp">
    <jsp:param name="active" value="home"/>   <%-- 파라미터 전달 --%>
</jsp:include>
```
- 요청 시점에 대상 페이지를 **따로 실행한 결과**를 끼워 넣는다.
- 디렉티브 `<%@ include %>`(번역 시 합침)와의 차이:

| | `<%@ include %>` (디렉티브) | `<jsp:include>` (액션) |
|---|---|---|
| 시점 | 번역(컴파일) 시 | 실행(요청) 시 |
| 방식 | 소스를 그대로 합침 | 실행 결과를 끼움 |
| 변수 공유 | O (한 파일이 됨) | X (별도 실행) |
| 적합 | 고정 헤더/푸터 | 자주 바뀌는 조각·파라미터 전달 |

#### (2) `<jsp:forward>` — 다른 페이지로 제어 이동

```jsp
<jsp:forward page="login.jsp"/>
```

#### (3) `<jsp:useBean>` / `setProperty` / `getProperty` — 자바빈 사용

```jsp
<%-- id 이름으로 빈 생성(또는 기존 것 사용) --%>
<jsp:useBean id="member" class="com.example.MemberDTO" scope="request"/>

<%-- 요청 파라미터를 같은 이름 프로퍼티에 자동 매핑 --%>
<jsp:setProperty name="member" property="*"/>
<jsp:setProperty name="member" property="age" value="20"/>

<%-- 프로퍼티 값 출력 --%>
이름: <jsp:getProperty name="member" property="name"/>
```
> 이 빈 액션들도 MVC(스프링)에서는 거의 안 쓴다. Controller가 객체를 만들어 model에 담는 방식으로 대체된다.

### 3-6. 스코프(scope) — 데이터가 살아있는 범위

내장 객체에 `setAttribute`로 담은 값은 **스코프**에 따라 생존 기간이 다르다. EL이 값을 찾을 때도 이 순서를 탄다.

| 스코프 | 객체 | 살아있는 범위 |
|--------|------|---------------|
| page | pageContext | 현재 JSP 페이지 안에서만 |
| request | request | 한 번의 요청~응답 (forward 포함) |
| session | session | 같은 브라우저의 세션 동안 (로그인 유지 등) |
| application | application | 앱 시작~종료, 모든 사용자 공유 |

```jsp
<% request.setAttribute("msg", "hello"); %>   <%-- request 스코프에 저장 --%>
${msg}                                         <%-- EL이 page→request→... 순으로 찾아 출력 --%>
${requestScope.msg}                            <%-- 특정 스코프 콕 집어 접근 --%>
```
- 스프링 Controller의 `model.addAttribute(...)` 는 보통 **request 스코프**에 담긴다.

### 3-7. 에러 페이지 처리

```jsp
<%-- 일반 페이지: 예외 나면 error.jsp 로 보냄 --%>
<%@ page errorPage="error.jsp" %>
```
```jsp
<%-- error.jsp: 에러 전용으로 선언하면 exception 객체 사용 가능 --%>
<%@ page isErrorPage="true" %>
<h2>오류가 발생했습니다</h2>
<p>${pageContext.exception.message}</p>
```
> 실무에선 `web.xml`의 `<error-page>` 나 스프링의 `@ExceptionHandler` 로 통합 처리하는 경우가 많다.

## 4. 데이터 전달 — Controller의 model → JSP

Controller에서 `model`에 담은 값이 JSP에서 `${이름}`(EL)로 꺼내진다.

```java
// BoardController.java
@GetMapping("/list")
public void list(@ModelAttribute("cri") Criteria cri, Model model) {
    List<BoardVO> list = boardService.getList(cri);
    model.addAttribute("list", list);   // ← "list" 라는 이름으로 담음
}
```

```jsp
<%-- list.jsp : 같은 이름 "list"로 꺼냄 --%>
<c:forEach var="board" items="${list}">
    <td>${board.bno}</td>
    <td>${board.title}</td>
</c:forEach>
```

핵심: **담을 때 쓴 이름 = 꺼낼 때 쓰는 이름.** (위 예에서 `"list"`)

## 5. 실제 예제 — 게시판 목록 (`board/list.jsp`)

게시판 목록 화면을 단순화한 예제. 데이터 출력 + 페이지네이션까지 포함한다.

```jsp
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
<%@ include file="../includes/header.jsp" %>

<table class="table table-bordered">
    <thead>
        <tr><th>Bno</th><th>Title</th><th>Writer</th><th>RegDate</th></tr>
    </thead>
    <tbody>
        <%-- model에 담긴 list를 한 줄씩 반복 출력 --%>
        <c:forEach var="board" items="${list}">
            <tr data-bno="${board.bno}">
                <td><c:out value="${board.bno}"/></td>
                <td><c:out value="${board.title}"/></td>
                <td><c:out value="${board.writer}"/></td>
                <td><c:out value="${board.regDate}"/></td>
            </tr>
        </c:forEach>
    </tbody>
</table>

<%-- 페이지 번호 네비게이션: pageMaker(PageDTO)가 model에 있어야 동작 --%>
<ul class="pagination">
    <c:if test="${pageMaker.prev}">
        <li class="page-item">
            <a class="page-link" href="list?pageNum=${pageMaker.startPage - 1}">Previous</a>
        </li>
    </c:if>

    <c:forEach var="num" begin="${pageMaker.startPage}" end="${pageMaker.endPage}">
        <li class="page-item ${pageMaker.cri.pageNum == num ? 'active' : ''}">
            <a class="page-link"
               href="list?pageNum=${num}&amount=${pageMaker.cri.amount}">${num}</a>
        </li>
    </c:forEach>

    <c:if test="${pageMaker.next}">
        <li class="page-item">
            <a class="page-link" href="list?pageNum=${pageMaker.endPage + 1}">Next</a>
        </li>
    </c:if>
</ul>

<%@ include file="../includes/footer.jsp" %>
```

> 페이지네이션 계산 원리(`startPage` / `endPage` / `prev` / `next`)는 별도의 페이징 글 참고.

## 6. include — 화면 조각 재사용

머리말/꼬리말처럼 모든 페이지에 공통인 부분을 분리한다.

```jsp
<%@ include file="../includes/header.jsp" %>   <%-- 위쪽 공통 (메뉴, CSS 링크 등) --%>
   ... 페이지 고유 내용 ...
<%@ include file="../includes/footer.jsp" %>   <%-- 아래쪽 공통 --%>
```

- `<%@ include %>` (정적, **컴파일 시 합침**) vs `<jsp:include>` (동적, **실행 시 호출**).
- 헤더/푸터처럼 잘 안 바뀌는 조각은 보통 정적 include를 쓴다.

## 7. 자주 하는 실수 / 주의점

| 증상 | 원인 | 해결 |
|------|------|------|
| `${list}` 가 글자 그대로 출력됨 | EL이 꺼져 있거나 JSTL/서블릿 버전 문제 | `page` 디렉티브·라이브러리 의존성 확인 |
| 한글 깨짐 | 인코딩 미설정 | `contentType="text/html;charset=UTF-8"` |
| `${board.title}` 안 나옴 | model 이름 불일치 / getter 없음 | model 이름·DTO getter 확인 |
| jsp 수정했는데 반영 안 됨 | 이전 `.class` 캐시 | 서버 재시작 / 빌드 정리 |
| URL로 jsp 직접 접근 시 404 | `WEB-INF` 보호 영역 | 반드시 Controller 경유 |

## 8. 한 줄 요약

> JSP = **서버에서 서블릿으로 변환·실행되어 HTML을 만들어내는 View.**
> 자바 코드를 직접 박지 말고(`<% %>` 지양), Controller가 `model`에 담아준 값을
> **EL(`${}`) + JSTL(`<c:...>`)** 로 꺼내 화면을 그린다.
