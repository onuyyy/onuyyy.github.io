> JSP 자체의 동작원리·사용법은 별도의 JSP 글에서 다룬다. 이 문서는 JSP 안에서 쓰는
> **EL(표현 언어)** 과 **JSTL(태그 라이브러리)** 문법만 모았다.
> 둘은 짝이다: **EL = 값 꺼내기/계산**, **JSTL = 흐름 제어(반복·조건)**.

---

## 1. EL (Expression Language) — `${...}`

스크립트릿(`<%= %>`) 없이 **값을 간결하게 꺼내고 계산**하는 표현식.

### 1-1. 기본 사용

```jsp
${board.title}      <%-- board.getTitle() 을 호출 (getter 규칙) --%>
${list}             <%-- model에 담긴 list --%>
${10 + 5}           <%-- 15 (산술 연산도 가능) --%>
```

> 핵심: `${board.title}` 은 필드가 아니라 **getter(`getTitle()`)** 를 호출한다.
> 그래서 DTO에 getter(보통 Lombok `@Getter`/`@Data`)가 있어야 값이 나온다.

### 1-2. 값을 찾는 순서 (스코프)

이름만 쓰면 EL이 좁은 범위부터 넓은 범위로 자동 탐색한다.

```
page  →  request  →  session  →  application
```

- Controller의 `model.addAttribute("list", ...)` 는 보통 **request** 스코프에 담긴다.
- 같은 이름이 여러 스코프에 있으면 **가장 좁은(page)** 것이 우선.

### 1-3. 객체 접근 — `.` 과 `[]`

```jsp
${board.title}        <%-- 점 표기 --%>
${board["title"]}     <%-- 대괄호 표기 (동일 의미) --%>
${map["key"]}         <%-- Map: key로 접근 --%>
${list[0]}            <%-- List: 인덱스로 접근 --%>
```

### 1-4. 연산자

```jsp
${a + b}              <%-- 산술: + - * / %  --%>
${a == b}             <%-- 비교: == != < > <= >=  (eq ne lt gt le ge 도 가능) --%>
${a > 0 && b < 10}    <%-- 논리: && || !  (and or not 도 가능) --%>
${empty list}         <%-- list가 null 이거나 비었으면 true --%>
${cri.pageNum == num ? 'active' : ''}   <%-- 삼항 연산자 --%>
```

`empty` 는 실무에서 매우 자주 쓴다 (null·빈 컬렉션·빈 문자열을 한 번에 검사).

### 1-5. 내장 객체 (자주 쓰는 것)

```jsp
${param.pageNum}        <%-- 요청 파라미터 ?pageNum=3 → "3" --%>
${pageContext.request.contextPath}   <%-- 앱 컨텍스트 경로 (링크 절대경로용) --%>
```

```jsp
<%-- 링크에 contextPath 붙이는 패턴 (경로 깨짐 방지) --%>
<a href="${pageContext.request.contextPath}/board/list">목록</a>
```

## 2. JSTL (JSP Standard Tag Library)

반복·조건 같은 **흐름 제어**를 태그로 처리하는 표준 라이브러리.
스크립트릿(`<% if ... %>`)을 없애준다.

### 2-1. 사용 선언 (taglib)

JSP 맨 위에서 한 번 선언해야 `<c:...>` 를 쓸 수 있다.

```jsp
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
```

| prefix | uri | 용도 |
|--------|-----|------|
| `c` | `.../jstl/core` | 반복·조건·변수 (Core) — **가장 많이 씀** |
| `fmt` | `.../jstl/fmt` | 숫자·날짜 포맷, 다국어 |
| `fn` | `.../jstl/functions` | 문자열 함수 (length, contains 등) |

> ⚠️ 톰캣/서블릿 버전에 따라 uri가 `https://jakarta.ee/...` 로 바뀔 수 있다.
> 레거시(javax 기반) 환경에서는 보통 `http://java.sun.com/jsp/jstl/core` 를 사용한다.

### 2-2. `<c:out>` — 안전하게 출력

```jsp
<c:out value="${board.title}"/>
```

- HTML 특수문자(`<`, `>`, `&`)를 **이스케이프** → XSS 방지.
- 그냥 `${board.title}` 보다 안전. 사용자 입력 출력 시 권장.

### 2-3. `<c:forEach>` — 반복

```jsp
<%-- 컬렉션 반복 --%>
<c:forEach var="board" items="${list}">
    <tr>
        <td>${board.bno}</td>
        <td>${board.title}</td>
    </tr>
</c:forEach>

<%-- 숫자 범위 반복 (페이지 번호 출력에 사용) --%>
<c:forEach var="num" begin="${pageMaker.startPage}" end="${pageMaker.endPage}">
    <a href="list?pageNum=${num}">${num}</a>
</c:forEach>

<%-- 상태 객체(varStatus): 인덱스/순번/처음·끝 여부 --%>
<c:forEach var="board" items="${list}" varStatus="status">
    ${status.index}   <%-- 0부터 --%>
    ${status.count}   <%-- 1부터 --%>
    ${status.first}   <%-- 첫 번째면 true --%>
    ${status.last}    <%-- 마지막이면 true --%>
</c:forEach>
```

| 속성 | 의미 |
|------|------|
| `var` | 현재 항목을 담을 변수명 |
| `items` | 반복할 컬렉션 (List, 배열, Map...) |
| `begin` / `end` | 숫자 범위 반복 시 시작/끝 |
| `varStatus` | 반복 상태(index, count, first, last) |

### 2-4. `<c:if>` — 단일 조건

```jsp
<c:if test="${pageMaker.prev}">
    <a href="list?pageNum=${pageMaker.startPage - 1}">Previous</a>
</c:if>
```

- 조건이 true일 때만 내부 출력. **else가 없다** → else가 필요하면 `<c:choose>`.

### 2-5. `<c:choose>` — 다중 조건 (if-else if-else)

```jsp
<c:choose>
    <c:when test="${empty list}">
        <tr><td colspan="4">게시글이 없습니다.</td></tr>
    </c:when>
    <c:when test="${pageMaker.total > 100}">
        <p>게시글이 많습니다.</p>
    </c:when>
    <c:otherwise>
        <p>총 ${pageMaker.total} 건</p>
    </c:otherwise>
</c:choose>
```

### 2-6. `<c:set>` / `<c:url>`

```jsp
<c:set var="ctx" value="${pageContext.request.contextPath}"/>   <%-- 변수 정의 --%>
<a href="${ctx}/board/list">목록</a>

<c:url var="listUrl" value="/board/list">      <%-- 파라미터 안전하게 붙인 URL --%>
    <c:param name="pageNum" value="3"/>
</c:url>
<a href="${listUrl}">3페이지</a>
```

### 2-7. `<fmt:...>` — 날짜/숫자 포맷

```jsp
<%@ taglib prefix="fmt" uri="http://java.sun.com/jsp/jstl/fmt" %>

<fmt:formatDate value="${board.regDate}" pattern="yyyy-MM-dd HH:mm"/>
<fmt:formatNumber value="${pageMaker.total}" pattern="#,###"/>   <%-- 1,234 --%>
```

## 3. EL + JSTL 함께 쓰는 실전 예제 (게시판)

```jsp
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%@ taglib prefix="c"   uri="http://java.sun.com/jsp/jstl/core" %>
<%@ taglib prefix="fmt" uri="http://java.sun.com/jsp/jstl/fmt" %>

<table>
<c:choose>
    <%-- 결과가 없을 때 --%>
    <c:when test="${empty list}">
        <tr><td colspan="4">등록된 게시글이 없습니다.</td></tr>
    </c:when>
    <%-- 결과가 있을 때: 한 줄씩 반복 --%>
    <c:otherwise>
        <c:forEach var="board" items="${list}">
            <tr>
                <td>${board.bno}</td>
                <td><c:out value="${board.title}"/></td>
                <td><c:out value="${board.writer}"/></td>
                <td><fmt:formatDate value="${board.regDate}" pattern="yyyy-MM-dd"/></td>
            </tr>
        </c:forEach>
    </c:otherwise>
</c:choose>
</table>

<%-- 페이지 번호: EL 계산 + JSTL 반복/조건 조합 --%>
<ul class="pagination">
    <c:if test="${pageMaker.prev}">
        <li><a href="list?pageNum=${pageMaker.startPage - 1}">Prev</a></li>
    </c:if>
    <c:forEach var="num" begin="${pageMaker.startPage}" end="${pageMaker.endPage}">
        <li class="${pageMaker.cri.pageNum == num ? 'active' : ''}">
            <a href="list?pageNum=${num}">${num}</a>
        </li>
    </c:forEach>
    <c:if test="${pageMaker.next}">
        <li><a href="list?pageNum=${pageMaker.endPage + 1}">Next</a></li>
    </c:if>
</ul>
```

## 4. 자주 하는 실수

| 증상 | 원인 | 해결 |
|------|------|------|
| `${...}` 가 글자 그대로 출력 | EL 비활성 / 라이브러리 누락 | `page`·taglib·의존성 확인 |
| `<c:forEach>` 가 안 먹힘 | taglib 선언 누락 | 상단 `<%@ taglib ... %>` 추가 |
| `${board.title}` 빈값 | getter 없음 / 이름 오타 | DTO getter·model 이름 확인 |
| 날짜가 이상하게 출력 | 포맷 미적용 | `<fmt:formatDate>` 사용 |
| 사용자 입력에 `<script>` 실행됨 | 이스케이프 안 함 | `${}` 대신 `<c:out>` |

---

## 5. 한 줄 요약

> **EL(`${}`)** 은 model 값을 **꺼내고 계산**, **JSTL(`<c:...>`)** 은 **반복·조건**.
> 이 둘로 스크립트릿(`<% %>`) 없이 깔끔한 View를 만든다.
