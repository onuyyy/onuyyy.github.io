## 1. DOM 이란?

**DOM(Document Object Model)** 은 브라우저가 HTML 문서를 읽어 **메모리에 만든 객체 트리**다.
즉 HTML 태그 하나하나를 JavaScript에서 접근·조작할 수 있게 만든 **인터페이스(실물 객체)** 다.

- **HTML** : 파일에 쓴 텍스트(설계도). 그냥 문자열.
- **DOM** : 브라우저가 그 텍스트를 파싱해서 만든 객체 트리(실제로 지은 집).

```
HTML(텍스트)  ──(브라우저 파싱)──►  DOM(객체 트리)  ◄──(조작)── JavaScript
```

> `document.querySelector(...)`, `element.append(...)` 처럼 점 찍어 다루는 건 전부 DOM 객체를 만지는 것이다.
> HTML 텍스트를 직접 고치는 게 아니라, 브라우저가 만들어둔 객체를 바꾸면 화면이 다시 그려진다.

## 2. 렌더링 순서 — HTML이 화면이 되기까지

브라우저가 HTML/CSS를 받아 픽셀로 그리는 과정을 **Critical Rendering Path** 라 한다.

```
HTML  ──파싱──►  DOM 트리 ─┐
                            ├──► Render Tree ──► Layout(배치) ──► Paint(그리기) ──► 화면
CSS   ──파싱──►  CSSOM 트리 ┘                     (Reflow)         (Repaint)
```

| 단계 | 하는 일 |
|------|---------|
| DOM 생성 | HTML 텍스트 → 노드 객체 트리 |
| CSSOM 생성 | CSS 텍스트 → 스타일 객체 트리 |
| Render Tree | DOM + CSSOM 결합 (화면에 보이는 것만, `display:none` 제외) |
| Layout(Reflow) | 각 요소의 위치·크기 계산 |
| Paint(Repaint) | 실제 픽셀 칠하기 |

- **Reflow(레이아웃 재계산)** : 크기·위치가 바뀔 때. 비쌈. (`width`, `display`, DOM 추가/삭제 등)
- **Repaint(다시 칠하기)** : 색·배경처럼 배치는 그대로일 때. Reflow보다 쌈. (`color`, `visibility` 등)
- 그래서 DOM 조작은 **모아서 한 번에** 하는 게 성능에 좋다 (→ 13장 최적화).

### 스크립트 실행 시점 — `<script>` 위치가 중요한 이유

```
<head> 안 <script>  : DOM이 아직 안 만들어진 상태 → querySelector가 null 반환 위험
</body> 직전 <script> : DOM 다 만들어진 뒤 실행 → 안전 (전통적 권장 위치)
```

- `DOMContentLoaded` 이벤트 : DOM 트리 완성 시점 (이미지 로딩은 안 기다림).
- `load` 이벤트 : 이미지·CSS 등 모든 리소스까지 로딩 완료 시점.

```javascript
document.addEventListener("DOMContentLoaded", () => {
  // 여기서 DOM 조작하면 안전
});
```

## 3. DOM 트리와 노드 종류

DOM 트리는 **노드(Node)** 들의 부모-자식 관계로 이뤄진다.

| 노드 종류 | 설명 | `nodeType` |
|-----------|------|:----------:|
| 문서 노드 (Document) | 트리 최상위 진입점. `document` | 9 |
| 요소 노드 (Element) | HTML 태그. `<div>`, `<tr>` 등 | 1 |
| 속성 노드 (Attribute) | 요소의 속성. `id="..."`, `data-bno="..."` | 2 |
| 텍스트 노드 (Text) | 실제 글자 내용. 트리의 **최하단** | 3 |

```
document (문서)
└─ <html>
   └─ <body>
      └─ <tr data-bno="10">   ← 요소 노드 (+ 속성 노드 data-bno)
         └─ "제목입니다"        ← 텍스트 노드 (요소의 자식)
```

> 헷갈림 주의 : 태그 안의 글자도 별도의 **텍스트 노드**다. 그래서 `childNodes`로 보면 눈에 안 보이는 공백·줄바꿈 텍스트 노드까지 잡힌다.

## 4. 요소 선택 (Selecting)

| 메서드 | 반환값 | 설명 |
|--------|--------|------|
| `getElementById('id')` | 단일 요소 | id로 1개 |
| `querySelector('css')` | 단일 요소 | CSS 선택자로 **첫 번째** 1개 |
| `querySelectorAll('css')` | **NodeList** | CSS 선택자로 전부 (정적) |
| `getElementsByClassName('cls')` | **HTMLCollection** | 클래스명으로 전부 (실시간) |
| `getElementsByTagName('tag')` | **HTMLCollection** | 태그명으로 전부 (실시간) |

```javascript
const one  = document.querySelector(".tbody");        // 첫 번째 .tbody
const many = document.querySelectorAll("tr");         // 모든 tr (NodeList)
```

### HTMLCollection vs NodeList — 꼭 구분할 것

| 특성 | HTMLCollection | NodeList |
|------|:--------------:|:--------:|
| 동적 업데이트 | O (실시간) | X (정적)¹ |
| `forEach` | ✕ | O |
| 만들어지는 곳 | `getElements*` | `querySelectorAll` |

¹ `childNodes`만 예외로 동적.

```javascript
// HTMLCollection은 forEach 불가 → 스프레드로 배열 변환 후 순회
const elems = document.getElementsByClassName("red");
[...elems].forEach(el => el.classList.replace("red", "blue"));
```

## 5. 요소 탐색 (Traversing)

`Node`로 끝나는 건 **텍스트 노드 포함**, `Element`로 끝나는 건 **요소만**.

| 텍스트 노드 포함 (Node) | 요소만 (Element) | 방향 |
|------------------------|------------------|------|
| `parentNode` | `parentElement` | 부모 |
| `childNodes` (NodeList) | `children` (HTMLCollection) | 자식 전체 |
| `firstChild` | `firstElementChild` | 첫 자식 |
| `lastChild` | `lastElementChild` | 마지막 자식 |
| `nextSibling` | `nextElementSibling` | 다음 형제 |
| `previousSibling` | `previousElementSibling` | 이전 형제 |

> 실무에선 공백 텍스트 노드에 안 걸리는 **`children` / `*ElementSibling` / `parentElement`** 쪽을 주로 쓴다.

### `closest()` — 위로 올라가며 가장 가까운 조상 찾기

```javascript
e.target.closest("tr");  // 클릭 지점에서 위로 올라가며 가장 가까운 tr
```

> 예를 들어 목록 화면(`list.jsp`)에서 "행 안의 글자를 클릭해도 그 행(`tr`) 전체를 잡기" 위해 쓴다.

## 6. 속성(Attribute) 다루기

```javascript
el.hasAttribute("data-bno")        // 존재 여부 (true/false)
el.getAttribute("data-bno")        // 값 읽기
el.setAttribute("action", "/board/read/10")  // 값 설정
el.removeAttribute("disabled")     // 제거
```

### `dataset` — `data-*` 커스텀 속성 전용

```html
<tr data-bno="10"></tr>
```
```javascript
tr.dataset.bno;   // "10"  ← data- 떼고 camelCase로 접근
```

> `list.jsp`의 `target.dataset.bno`가 바로 이것. 게시글 번호를 읽어온다.

## 7. 스타일 다루기

```javascript
el.style.color = "red";          // 인라인 스타일 직접 변경
el.style.display = "none";

// 적용된 최종 스타일 읽기 (계산된 값)
window.getComputedStyle(el).getPropertyValue("width");
```

> `el.style.xxx` 는 **인라인 스타일만** 읽고 쓴다. CSS 파일에 정의된 값은 `getComputedStyle`로 읽어야 한다.

## 8. 클래스(class) 다루기

```javascript
// className — 문자열 통째로 교체 (기존 클래스 다 날아감, 비권장)
el.className = "btn active";

// classList — 메서드 방식 (권장)
el.classList.add("active");          // 추가
el.classList.remove("active");       // 제거
el.classList.toggle("active");       // 있으면 빼고 없으면 넣고
el.classList.contains("active");     // 있는지 확인 (true/false)
el.classList.replace("old", "new");  // 교체
```

## 9. 콘텐츠 변경

| 프로퍼티 / 메서드 | 마크업 해석 | 특징 |
|-------------------|:-----------:|------|
| `innerHTML` | O | 자식 태그 포함 HTML. **XSS 위험** |
| `outerHTML` | O | 자기 자신까지 포함한 HTML |
| `textContent` | X | 순수 텍스트만. 태그도 글자로 출력. **안전** |
| `insertAdjacentHTML(pos, html)` | O | 원하는 위치에 HTML 삽입. **XSS 위험** |

```javascript
el.textContent = "<b>안녕</b>";   // 화면에 <b>안녕</b> 글자 그대로 출력
el.innerHTML  = "<b>안녕</b>";    // "안녕"이 굵게 렌더링됨

// insertAdjacentHTML 삽입 위치
// <!-- beforebegin --> <div> <!-- afterbegin --> ... <!-- beforeend --> </div> <!-- afterend -->
el.insertAdjacentHTML("beforeend", "<p>추가</p>");
```

> ⚠️ 사용자 입력을 `innerHTML`에 그대로 넣으면 **XSS(크로스 사이트 스크립팅)** 에 취약하다.
> 텍스트는 `textContent`, 요소는 아래 `createElement` 방식으로 넣는 게 안전.

## 10. 요소 생성·추가

```javascript
// 1) 생성
const li   = document.createElement("li");
const text = document.createTextNode("항목");  // (또는 li.textContent = "항목")
li.appendChild(text);

// 2) 추가
parent.appendChild(li);            // 마지막 자식으로
parent.append(li);                 // 마지막 자식으로 (여러 개·문자열도 OK)
parent.prepend(li);                // 첫 자식으로
parent.insertBefore(li, refNode);  // refNode 앞에 삽입
refNode.before(li);                // 형제로 앞에
refNode.after(li);                 // 형제로 뒤에
```

```javascript
// 여러 요소 한 번에 앞쪽으로 이동
const items = document.querySelectorAll(".selected");
document.querySelector("#newfriends").prepend(...items);
```

> **이동도 된다** : 이미 트리에 있는 노드를 `appendChild`하면 복사가 아니라 **그 위치로 이동**한다.

## 12. 이벤트 (Event)

```javascript
target.addEventListener("click", function (e) {
  // e = 이벤트 객체. 브라우저가 자동으로 첫 인자에 넘겨줌
  // e.target  : 실제로 클릭된 요소
  // e.currentTarget : 리스너가 붙은 요소
}, false);
//   ▲ 세 번째 인자: false(기본)=버블링 단계 / true=캡처 단계. 보통 생략
```

### 캡처링 & 버블링 — 이벤트 흐름 3단계

이벤트는 클릭된 요소에서 곧장 끝나지 않는다. **`document`에서 출발해 타깃까지 내려갔다가(캡처), 다시 `document`로 올라온다(버블).**

```html
<div id="grandpa">
  <div id="parent">
    <button id="child">클릭</button>
  </div>
</div>
```

`child`를 클릭하면:

```
        ① 캡처링 (위→아래로 내려감)
   document → grandpa → parent → child
                                   │
                                   ▼  ② 타깃 (목적지 도착)
                                  child
                                   │
        ③ 버블링 (아래→위로 올라감)
   document ← grandpa ← parent ← child
```

| 단계 | 이름 | 방향 |
|------|------|------|
| ① | **캡처링(Capturing)** | 바깥 → 안쪽 (조상 → 타깃) |
| ② | **타깃(Target)** | 실제 클릭된 요소 도착 |
| ③ | **버블링(Bubbling)** | 안쪽 → 바깥 (타깃 → 조상) |

> 물속 거품(bubble)이 아래에서 위로 떠오르듯 안쪽→바깥으로 전파돼서 "버블링".

**세 번째 인자가 어느 단계에서 잡을지 정한다:**
```javascript
el.addEventListener("click", fn, false); // false(기본)=버블링 단계, true=캡처 단계
```
- 대부분 **`false`(버블링)** 를 쓰고, 기본값이라 **보통 생략**한다.

**실제 출력 순서 확인:**
```javascript
grandpa.addEventListener("click", () => console.log("grandpa")); // 버블(기본)
parent .addEventListener("click", () => console.log("parent"));
child  .addEventListener("click", () => console.log("child"));
// child 클릭 시 →  child  →  parent  →  grandpa   (안쪽→바깥)
// 셋 다 3번째 인자 true(캡처)면 →  grandpa → parent → child (반대)
```

### `e.target` vs `e.currentTarget`

| | 가리키는 것 |
|---|---|
| `e.target` | **실제로 클릭된** 요소 (이벤트가 시작된 곳, 예: `<td>`) |
| `e.currentTarget` | **리스너가 붙은** 요소 (`this`와 동일, 예: `.tbody`) |

```javascript
tbody.addEventListener("click", function (e) {
  e.target;        // 진짜 클릭한 td (매번 다름)
  e.currentTarget; // 항상 tbody (리스너 주인)
});
```

### 전파 멈추기 — `stopPropagation()`

```javascript
child.addEventListener("click", function (e) {
  e.stopPropagation(); // 여기서 멈춤 → parent, grandpa 리스너는 실행 안 됨
});
```

> ⚠️ `e.preventDefault()` 와 혼동 주의. 이건 **브라우저 기본 동작**(링크 이동, 폼 제출 등)을 막는 것이고 전파(버블링)와는 무관하다.

### 이벤트 위임 (Event Delegation) — 버블링을 활용한 핵심 패턴

자식마다 리스너를 다는 대신 **부모에 하나만** 달고, 버블링으로 올라온 이벤트를 `e.target`으로 구분한다.

```javascript
// tr 100개에 각각 달지 않고, 부모 .tbody 하나에만 등록
document.querySelector(".tbody").addEventListener("click", function (e) {
  const tr = e.target.closest("tr");   // 어떤 행이 클릭됐는지 (e.target에서 위로)
  if (!tr) return;
  const bno = tr.dataset.bno;
  // ... 상세 페이지로 이동
});
```

이게 되는 원리 = **버블링**:
```
tr 안의 <td> 글자 클릭
  → td에서 click 발생
  → 버블링으로 위로: td → tr → tbody
  → tbody에 붙은 리스너가 올라온 이벤트를 잡음!
  → "어디서 시작됐나"는 e.target 으로 판별 (e.target.closest("tr"))
```

장점:
- 리스너 1개 → **메모리 절약·성능 향상**.
- 나중에 **동적으로 추가된 행**에도 자동으로 동작 (부모가 듣고 있으니까).

> 예를 들어 게시판 목록(`list.jsp`)의 클릭 핸들러가 정확히 이 이벤트 위임 패턴이다.

## 13. 성능 최적화 팁

1. **DOM 접근을 줄여라** — 반복문 안에서 매번 `querySelector` 하지 말고 변수에 캐싱.
2. **Reflow를 모아라** — 여러 번 추가할 땐 `DocumentFragment`에 모았다가 한 번에 append.
   ```javascript
   const frag = document.createDocumentFragment();
   data.forEach(d => {
     const li = document.createElement("li");
     li.textContent = d;
     frag.appendChild(li);     // 아직 화면 reflow 안 일어남
   });
   list.appendChild(frag);     // 여기서 한 번만 reflow
   ```
3. **`classList` 사용** — `className` 문자열 조작보다 안전·명확.
4. **`textContent` 우선** — 보안(XSS)상 `innerHTML`보다 안전.
5. **이벤트 위임** — 요소 많을 때 리스너 하나로 처리(12장).
6. **HTMLCollection 주의** — 실시간 갱신이라 반복 중 변하면 꼬임 → `[...el]`로 배열 변환 후 순회.

## 한눈 요약

- **DOM** = HTML(텍스트)을 브라우저가 만든 **객체 트리**. JS는 이걸 조작한다.
- 선택 : `querySelector` / `querySelectorAll`(NodeList) / `getElements*`(HTMLCollection)
- 탐색 : `children` / `parentElement` / `closest()`
- 속성 : `getAttribute`/`setAttribute`/`dataset`
- 클래스 : `classList.add/remove/toggle/contains`
- 콘텐츠 : `textContent`(안전) vs `innerHTML`(XSS 주의)
- 생성/추가 : `createElement` → `append`/`prepend`/`before`/`after`
- 이벤트 : `addEventListener`, 캡처/버블, **이벤트 위임**
- 성능 : DOM 접근 줄이고, reflow 모으고, fragment 활용
